## Summary

Add a local-only (not CI-wired) Grafana exploration stack layered on top of the existing `deploy/e2e/compose.single.yaml` e2e fixture, so a developer can browse real logs/metrics/traces flowing through pulsusdb in Grafana without hand-rolling a compose stack each time.

## Scope

- `deploy/e2e/grafana/provisioning/datasources/{tempo,prometheus}.yaml` — new Tempo-compat and native-Prometheus Grafana datasources against pulsusdb (the Loki-compat datasource already existed).
- `deploy/e2e/firehose/` — vendored copy (not a submodule) of the `firehose` synthetic telemetry generator from `terraform-google-monitoring`'s `traffic-gen/loaders/firehose`, so the compose overlay has no external-repo checkout dependency.
- `deploy/e2e/compose.grafana.yaml` — new compose overlay: builds `firehose` locally, wires it to the otel-collector, configured for a scaled-down 20-host demo fleet running in real time (`SPEEDUP=1`, `WINDOW_START` pinned to "now" at bring-up).
- `deploy/e2e/otel-config.grafana.yaml` — dedicated otel-collector config for this overlay (raises the `otlp` receiver's gRPC max-recv-message-size to 16MiB, trims the CI differential-scenario reference-store fan-outs) — kept separate from `otel-config.single.yaml`, which is shared with CI's pinned differential scenarios and must not change behavior for the harness's own stack.
- `deploy/e2e/grafana/provisioning/dashboards/{dashboards.yaml,files/firehose.json}` — a dashboard covering log rate/stream, CPU/memory/network/checkout-request metrics by host/region, and a Tempo traces panel.
- `Makefile` — `make grafana-up` / `grafana-down` / `grafana-logs`, computing `WINDOW_START` fresh on every invocation.

## Explicitly out of scope

Not wired into `cargo run -p pulsus-e2e` or CI — this is a manual local demo stack, not a pinned/scenario-asserted fixture like the rest of `deploy/e2e/`.

## Notes for reviewers

- `otel-config.single.yaml` (CI-shared) is untouched; a dedicated `otel-config.grafana.yaml` exists instead so this overlay can raise limits/trim pipelines without affecting CI's differential scenarios.
- Verified end-to-end locally: logs, metrics (`system_cpu_utilization`, `app_checkout_request_count`, etc.), and traces all confirmed flowing through pulsusdb and queryable via Grafana's own datasource proxy.
