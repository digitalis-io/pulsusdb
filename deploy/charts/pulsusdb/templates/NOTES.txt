Bundled ClickHouse ({{ .Values.topology }} topology) is a CI/compose-
fixture-parity deployment, not a hardened production ClickHouse — for
production, set `clickhouse.enabled=false` and point
`pulsusdb.config.clickhouse.server` at your own cluster (see README.md).

NOTE: if you rotate the bundled ClickHouse's generated password
out-of-band (e.g. `kubectl edit secret {{ include "pulsusdb.clickhouseSecretName" . }}`),
existing pods do NOT automatically restart — Helm's checksum-triggered
rollout mechanism cannot see live Secret content, only the values that
decide which Secret/keys are wired in (a documented Helm limitation, not
a chart bug). Run `kubectl rollout restart` on every pulsusdb and
ClickHouse workload in this release after an out-of-band rotation.
